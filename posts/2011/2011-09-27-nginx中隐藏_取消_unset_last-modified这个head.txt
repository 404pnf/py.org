# nginx中隐藏 取消 unset last-modified这个header

[tags]nginx[/tags] 内核的header我总弄不成功 http://wiki.nginx.org/NginxHttpHeadersModule 因为add\_header只是增加header，不能改写已有的 add\_header syntax: add\_header name value default: none context: http, server, location Directive adds headers to the HTTP response when the response code is equal to 200, 204, 301, 302 or 304. In the value it is possible to use variables. Note that it just appends a new header entry to the output header list. So you can't use this directive to rewrite existing headers like Server. Use the headers\_more module for it. ## 因此要用我们国人写的用模块 ：） ngx\_headers\_more - Set and clear input and output headers...more than "add"! http://wiki.nginx.org/NginxHttpHeadersMoreModule
